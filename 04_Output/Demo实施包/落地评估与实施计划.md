# Demo 落地评估与实施计划

## 结论

MiniMax3 的 Demo 方向可以落地，但表达重心需要上调：不能讲成“诊前、诊中、诊后三个功能依次执行”，而要讲成医疗 Copilot Agent 围绕 1 名高血压复诊模拟患者维护 Patient Memory、动态更新风险画像，并把诊后反馈回流到下一次诊前。

最大创新点应明确为：诊后反馈回流改变下一次诊前。三大 Workflow 是 Agent 可调度的能力，不是 Demo 的主角。

## 需要修正的点

1. 不把 Dify 写成唯一可运行路径。
   - 主路径仍用 Dify 云端版。
   - 兜底路径保留静态材料和录屏：数据、提示词、知识库片段、演示脚本都必须脱离 Dify 也能讲清。

2. RAG 来源必须固定。
   - 高血压阈值、分级、随访建议使用《中国高血压防治指南（2024年修订版）》作为主来源。
   - 药品内容以药品说明书为演示知识来源，具体实施前由人工核对。
   - Demo 输出必须写“辅助建议，不构成诊断或处方”。

3. 诊中输出不能生成“最终诊断”口吻。
   - 建议用“医生可核对要点”“可考虑的管理方向”“医嘱草稿”。
   - 最终诊疗决策必须在医生确认节点后才进入诊后任务。

4. 诊后 Workflow 不能暗示自动调整医嘱。
   - Agent 只做任务拆解、执行记录、异常标记、反馈摘要。
   - 漏服、血压升高、头晕加重等事件只触发医生确认，不自动改药。

5. 14 天排期需要前置数据冻结。
   - 7/20 前至少冻结主病例 JSON。
   - 7/21 前冻结知识库 v1。
   - 7/23 前冻结工具 schema 和三段提示词。
   - 7/25 若 Dify 原型未跑通，立刻切换到“三个独立应用 + 人工承接上下文”的兜底路径。

6. 强化 Agent 架构，而不是流程机器人。
   - 三大 Workflow 是 Agent 调用的能力，不是 Demo 的主角。
   - Patient Memory 保存医生确认后的可信上下文。
   - 风险画像提示医生关注优先级，但不替代临床判断。
   - 诊后反馈回流必须作为演示高潮，而不是诊后任务列表的附属结果。

## 实施资产清单

本实施包已补齐以下文件：

- `data/patient_wang_001.json`：主病例数据。
- `data/patient_memory_wang_001.json`：Patient Memory 示例。
- `data/risk_profile_wang_001.json`：风险画像示例。
- `data/followup_tasks.jsonl`：医生确认后生成的诊后任务。
- `data/followup_events.jsonl`：7 天患者反馈事件流。
- `data/knowledge_hypertension_demo.md`：高血压 Demo RAG 知识片段。
- `dify/tool_schemas.json`：模拟工具接口 schema。
- `dify/prompts.md`：三大 Workflow 与 HITL 输出模板。
- `dify/build_steps.md`：Dify 云端搭建步骤和兜底路径。
- `Agent架构强化说明.md`：Agent、Patient Memory、风险画像和反馈回流的表达框架。
- `演示脚本.md`：15 分钟演示流程。
- `验收清单.md`：实施前检查表。

## 推荐实施顺序

1. 在 Dify 创建知识库，导入 `data/knowledge_hypertension_demo.md`。
2. 在 Dify 创建 Chatflow/Workflow，按 `dify/build_steps.md` 配置三个阶段。
3. 将 `dify/prompts.md` 中的提示词分别放入诊前、诊中、诊后节点。
4. 将 `data/patient_wang_001.json` 作为模拟患者输入。
5. 将 `data/patient_memory_wang_001.json` 和 `data/risk_profile_wang_001.json` 作为 Agent 上下文资产。
6. 将 `data/followup_tasks.jsonl` 和 `data/followup_events.jsonl` 用作诊后演示脚本。
7. 按 `演示脚本.md` 做一次完整彩排，并用 `验收清单.md` 打勾。

## 最小可交付版本

如果时间不足，保留以下内容即可参加演示：

- 诊前：Patient Memory + 结构化摘要 + 缺失信息问题。
- 诊中：风险画像 + RAG 来源片段 + 医嘱草稿 + 医生确认。
- 诊后：预置任务 + 预置反馈 + 异常标记 + Memory 更新 + 下一次诊前反馈摘要。

砍掉以下增强项：

- 备用病例。
- 自定义时间轴页面。
- 复杂状态机。
- 真实工具服务端。
- 多患者管理。

## 医疗安全声明

本 Demo 全部使用虚构模拟数据，仅用于比赛原型演示和流程验证，不用于真实诊疗、诊断、处方或患者管理。所有涉及诊疗方向的内容必须由医生确认，AI 输出只作为信息整理和辅助参考。
