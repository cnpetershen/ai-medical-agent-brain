# Demo 验收清单

## 数据

- [ ] 主病例 `patient_wang_001.json` 可被打开，字段完整。
- [ ] Patient Memory `patient_memory_wang_001.json` 可被打开，能区分基线记忆、就诊记忆、决策记忆、执行记忆和回流记忆。
- [ ] 风险画像 `risk_profile_wang_001.json` 可被打开，且明确“不代表真实临床风险分层”。
- [ ] 近期血压记录至少 7 条，能支撑“近两周偏高”的叙事。
- [ ] 随访任务不少于 5 条，覆盖用药、监测、生活方式、复查、随访。
- [ ] 反馈事件不少于 7 条，至少包含 1 个漏服和 1 个症状/血压异常。
- [ ] 所有数据均标注为虚构模拟数据。

## RAG

- [ ] 知识库已导入 Dify。
- [ ] 诊中输出至少引用 2 条知识片段。
- [ ] 来源名称显示在建议旁边。
- [ ] 未检索到依据时不会编造来源。

## HITL

- [ ] 诊前摘要有医生确认区。
- [ ] 诊中医嘱草稿有医生确认区。
- [ ] 诊后异常摘要有医生确认区。
- [ ] 未经医生确认的医嘱不会进入诊后任务。
- [ ] 未经医生确认的 AI 输出不会写入可信 Patient Memory。

## Agent 架构

- [ ] 演示中能讲清 Agent 不是流程机器人。
- [ ] 三大 Workflow 被表述为 Agent 调度的能力，而不是三个孤立功能。
- [ ] 诊前能展示 Patient Memory 被读取。
- [ ] 诊中能展示风险画像参与医生关注点排序。
- [ ] 诊后能展示 Patient Memory 与风险画像被更新。

## 安全边界

- [ ] 输出不出现“最终诊断由 AI 确定”。
- [ ] 输出不出现“AI 自动开方/改药”。
- [ ] 输出明确为辅助草稿。
- [ ] 诊后只做跟踪、异常标记和反馈回流。
