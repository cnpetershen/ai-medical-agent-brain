# 15 分钟医疗旅程演示脚本

## 0:00-1:00 开场

本 Demo 不展示功能列表，只展示王某的一次完整医疗旅程。医疗 Copilot Agent 不是流程机器人，而是围绕同一名患者持续维护 Patient Memory、风险画像和医生确认记录。最大创新点是：诊后反馈会回流并改变下一次诊前。

## 1:00-4:00 诊前：Agent 读取患者记忆

输入患者口语化描述：

```text
医生您好，我最近两周早上量血压经常一百五六十，有时候头晕。药还是每天吃一片氨氯地平，但有两天忘了吃，最近工作忙睡得也少。
```

展示：

- Agent 读取 Patient Memory：高血压 3 年、当前氨氯地平用药、既往复诊计划。
- Agent 读取风险画像：近期血压控制、漏服、头晕和生活方式风险。
- Copilot 调用模拟患者档案和近期血压。
- 输出结构化诊前摘要。
- 自动列出缺失信息和补充问题。
- 医生执行“采纳 / 修改 / 拒绝”确认。

讲解重点：诊前不是填表，而是 Agent 把患者过去、现在和风险线索组织给医生。

## 4:00-8:00 诊中：医生确认下的辅助决策

医生输入：

```text
请基于已确认诊前摘要和高血压指南，整理复诊管理要点，并生成供我审核的医嘱草稿。
```

展示：

- Copilot 检索高血压知识库。
- Copilot 把 RAG 来源与王某的风险画像对齐。
- 输出带来源的管理要点。
- 生成医嘱草稿。
- 标出“哪些确认项将写回 Patient Memory”。
- 医生逐条确认，确认后才进入诊后。

讲解重点：Agent 不是替医生下结论，而是把患者上下文、风险线索和指南依据组织成可审核草稿。

## 8:00-12:00 诊后：反馈开始改变患者上下文

展示：

- 医生确认后的计划被拆解为用药、血压监测、生活方式、复查和随访任务。
- 模拟 7 天患者反馈事件流。
- Copilot 标记第 3 天头晕加重和第 4 天漏服。
- Copilot 生成风险画像更新草稿。
- Copilot 生成 Patient Memory 写回内容。

讲解重点：诊后不是“任务完成打勾”，而是患者真实执行情况进入连续照护上下文。AI 不修改医嘱，只做执行跟踪、风险提示和待医生确认的记忆更新。

## 12:00-14:00 闭环回流：最大创新点

展示下一次诊前摘要开头：

```markdown
上次诊后反馈：患者 7 天内完成多数血压记录，曾出现 1 次漏服；第 3 天报告头晕较前明显，后续减轻；家庭血压多数仍高于目标范围，建议医生复诊时重点核对依从性、测量方法、症状变化和是否需进一步检查。
```

继续展示风险画像变化：

```markdown
风险画像更新：血压控制风险保持中-高；用药依从性风险由中提升为需重点关注；症状变化风险需医生复核；连续照护断点风险下降，因为反馈已写入下一次诊前摘要。
```

讲解重点：这一步是 Demo 的主菜。诊后信息不是沉没数据，而是 Patient Memory 的新增证据，下一次医生接诊前就已经知道“这名患者上次回家后发生了什么”。

## 14:00-15:00 收束

强调三点：

- Agent 架构：Patient Memory + 风险画像 + RAG + Tool Calling + HITL。
- 医生主导：三处 HITL，关键内容确认后才写入 Memory。
- 最大创新：诊后反馈回流到下一次诊前，让一次就诊变成连续医疗旅程。
