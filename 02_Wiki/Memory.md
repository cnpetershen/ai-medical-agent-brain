# Memory

## 定义

Memory 是 [[医生AI Copilot]] 中用于维护患者上下文、医生偏好、Workflow 状态和历史任务记录的连续记忆能力。

在本项目中，Memory 的核心作用是支撑 [[连续照护闭环]]，让诊前、诊中、诊后的信息能够衔接，而不是每次任务都从零开始。

## 在医疗Copilot中的作用

- 保存患者在不同阶段产生的关键上下文。
- 记录诊前摘要、诊中确认事项和诊后执行反馈。
- 支持 [[医疗AI Agent]] 判断当前任务所处阶段。
- 帮助诊后反馈回流到下一次诊前信息采集。
- 保持医生确认过的信息可追溯，避免重复询问和重复整理。

## 为什么项目需要

连续照护的难点不只是单次就诊的信息处理，而是跨时间、跨阶段的信息衔接。

如果没有 Memory，[[诊前信息采集 Workflow]]、[[诊中辅助决策 Workflow]] 和 [[诊后医嘱执行 Workflow]] 之间会断裂，患者的诊后反馈也难以成为下一次诊疗的有效输入。

## 支撑哪个Workflow

- [[诊前信息采集 Workflow]]：使用历史记录、上次诊后反馈和近期健康数据生成诊前上下文。
- [[诊中辅助决策 Workflow]]：提供历史病历、医生确认事项和当前诊疗上下文。
- [[诊后医嘱执行 Workflow]]：记录医嘱任务状态、患者反馈和随访进展。

## 输入输出

输入：

- 患者基础信息和历史诊疗记录。
- 医生确认后的诊疗摘要和医嘱。
- Workflow 执行状态。
- 患者诊后任务反馈和健康数据。
- 医生偏好、常用处理方式或审核记录。

输出：

- 当前任务所需的患者上下文。
- 上一次诊疗和诊后管理摘要。
- Workflow 状态更新。
- 下一次诊前信息采集可使用的反馈材料。
- 医生可追溯的历史确认记录。

## 与其他知识节点关系

- [[连续照护闭环]]：Memory 支撑闭环跨阶段衔接。
- [[医生AI Copilot]]：Memory 是 Copilot 维持上下文的基础能力。
- [[医疗AI Agent]]：医疗AI Agent 依赖 Memory 判断上下文和任务状态。
- [[Agentic Workflow]]：Memory 记录 Workflow 的状态和结果。
- [[诊后医嘱执行 Workflow]]：Memory 保存诊后反馈并回流到后续流程。
- [[Human-in-the-loop]]：Memory 记录医生确认和审核结果。

## 备注

Memory 必须遵守患者隐私保护和最小必要原则。它不是无限保存所有数据，而是保存连续照护所需的关键上下文和可追溯记录。
