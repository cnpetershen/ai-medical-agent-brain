# 风险画像

## 定义

风险画像是 [[医生AI Copilot]] 围绕单个患者持续维护的关注优先级视图。

它基于 [[Memory|Patient Memory]]、患者健康数据、医嘱执行反馈和医生确认记录，帮助 [[医疗AI Agent]] 在诊前、诊中、诊后提示医生重点关注哪些风险线索。

风险画像不是诊断结论，不是临床风险分层替代品，也不自动触发治疗调整。

## 在本项目中的作用

- 将诊前信息、诊中确认事项和诊后反馈组织为可跟踪的风险线索。
- 帮助医生快速看到患者当前最需要核对的问题。
- 支撑诊后反馈回流，让下一次诊前不再从零开始。
- 防止 Demo 退化为功能清单或流程机器人，突出 Agent 对患者旅程的持续理解。

## 画像维度

本项目 Demo 中的风险画像至少包含：

| 维度 | 示例 | 用途 |
| --- | --- | --- |
| 血压控制风险 | 家庭血压多次偏高 | 提示医生核对测量方法、趋势和症状 |
| 用药依从性风险 | 漏服、误服、未理解用药计划 | 提示医生关注执行障碍 |
| 症状变化风险 | 头晕加重、胸闷、乏力等反馈 | 提示医生确认是否需要进一步处理 |
| 生活方式风险 | 高盐饮食、睡眠不足、运动不足 | 支撑可执行健康管理建议 |
| 连续照护断点风险 | 诊后反馈未回流、复诊资料缺失 | 提醒系统把诊后信息带回下一次诊前 |

## Agent 如何使用

[[医疗AI Agent]] 在不同阶段使用风险画像：

- 诊前：把上次诊后反馈和当前风险线索放入诊前摘要顶部。
- 诊中：结合 RAG 检索结果生成供医生核对的管理方向。
- 诊后：根据患者执行反馈生成风险变化草稿，并交由医生确认。
- 下一次诊前：把医生确认后的风险变化写入 [[Memory]]，形成连续照护闭环。

## 安全边界

- 风险画像只提示医生关注优先级。
- 风险画像不得生成最终诊断。
- 风险画像不得自动调整医嘱。
- 风险画像更新必须标明证据来源。
- 涉及异常、症状变化或医嘱处理的内容必须进入 [[Human-in-the-loop]]。

## 与其他知识节点关系

- [[Memory]]：风险画像从 Memory 中读取可信上下文，也将医生确认后的风险变化写回 Memory。
- [[连续照护闭环]]：风险画像帮助诊后反馈回流到下一次诊前。
- [[医疗AI Agent]]：Agent 使用风险画像组织上下文和调度 Workflow。
- [[诊后医嘱执行 Workflow]]：诊后反馈是风险画像更新的重要来源。
- [[Human-in-the-loop]]：医生确认风险提示是否进入可信上下文。

## 备注

风险画像的价值在于让医生更快看到“这名患者这次为什么需要重点关注”，而不是让 AI 替医生判断“应该怎么治疗”。
