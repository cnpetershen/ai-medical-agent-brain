# 医疗AI Agent

## 定义

医疗AI Agent 是本项目中 [[医生AI Copilot]] 的智能调度中枢。

它根据医生工作场景、患者上下文和任务状态，调度 [[诊前信息采集 Workflow]]、[[诊中辅助决策 Workflow]] 和 [[诊后医嘱执行 Workflow]]，辅助医生完成贯穿诊前、诊中、诊后的 [[连续照护闭环]]。

## 在医疗Copilot中的作用

- 理解医生在不同诊疗阶段提出的任务需求。
- 组织患者信息、历史记录、医嘱任务和随访反馈。
- 判断当前任务应进入哪个 [[Agentic Workflow]]。
- 协调 [[RAG]]、[[Memory]]、[[Tool Calling]] 等能力为 Workflow 服务。
- 在关键节点交由 [[Human-in-the-loop]] 机制进行医生审核。

## 为什么项目需要

本项目不是单点 AI 工具，而是医生工作流程中的连续协同系统。

如果没有医疗AI Agent，诊前信息、诊中辅助和诊后管理会停留在分散功能中，难以形成统一调度。医疗AI Agent 的价值在于把多个 Workflow 和多种 AI 能力组织到同一条医生主导的照护链路中。

## 支撑哪个Workflow

- [[诊前信息采集 Workflow]]：调度信息收集、结构化整理和缺失信息提示。
- [[诊中辅助决策 Workflow]]：调度知识检索、检查结果整理和病历摘要生成。
- [[诊后医嘱执行 Workflow]]：调度医嘱任务拆解、执行跟踪和随访反馈回流。

## 输入输出

输入：

- 医生任务指令。
- 患者诊前信息。
- 历史诊疗记录。
- 检查结果和健康数据。
- 医嘱、随访和任务状态。

输出：

- 应执行的 Workflow。
- 医生可审核的摘要、提示或任务草稿。
- 需要调用的工具或数据接口请求。
- 需要医生确认的关键节点。
- 可回流到连续照护流程的上下文更新。

## 与其他知识节点关系

- [[医生AI Copilot]]：医疗AI Agent 是其内部智能调度中枢。
- [[Agentic Workflow]]：医疗AI Agent 通过 Workflow 约束执行行为。
- [[连续照护闭环]]：医疗AI Agent 服务的业务目标。
- [[RAG]]：为诊中知识辅助提供外部知识连接。
- [[Memory]]：为连续照护提供跨阶段上下文。
- [[Tool Calling]]：连接外部医疗系统和任务工具。
- [[Human-in-the-loop]]：保证医生审核和医疗安全。

## 备注

本节点只描述项目中的医疗AI Agent，不讨论通用 Agent 框架。医疗AI Agent 不替代医生，不自动诊断，只负责在医生主导下调度和协同。
