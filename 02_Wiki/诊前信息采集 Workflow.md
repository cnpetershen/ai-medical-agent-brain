# 诊前信息采集 Workflow

## 定义

诊前信息采集 Workflow 是 [[医生AI Copilot]] 在患者就诊前运行的信息整理流程。

它通过 [[Agentic Workflow]] 将患者基础信息、症状描述、历史病历和就诊准备事项整理为医生可快速查看的诊前上下文，服务于 [[连续照护闭环]] 的起点。

## 临床场景

患者正式进入医生诊疗前，往往已经产生大量分散信息，例如主诉、现病史、既往史、用药情况、检查资料和本次就诊诉求。

该 Workflow 面向的场景包括：

- 患者预约或报到后，系统收集基础信息和症状描述。
- 医生接诊前，需要快速了解患者本次就诊背景。
- 慢病、复诊或连续管理患者需要汇总近期健康状态。
- 患者提供的信息不完整，需要提示补充关键问题。

## 解决的问题

- 患者信息分散，医生接诊前需要额外花时间查阅和追问。
- 患者自由描述不够结构化，难以直接进入诊疗判断。
- 历史病历、检查记录和近期健康数据之间缺少就诊前整合。
- 诊前准备不足会增加诊中沟通成本，影响后续连续照护。

## 工作流程

1. 接收患者基础信息、主诉、症状描述和历史资料。
2. 识别本次就诊相关的关键信息和缺失信息。
3. 将患者自由描述整理为结构化诊前摘要。
4. 关联既往病史、近期用药、检查记录和随访反馈。
5. 生成医生接诊前可查看的就诊准备材料。
6. 对需要医生关注或患者补充的信息进行标记。

## 输入数据

- 患者基础信息。
- 本次就诊主诉和症状描述。
- 既往病史、过敏史、用药史。
- 历史诊疗记录或电子病历摘要。
- 近期健康数据，例如血压、血糖、服药反馈。
- 患者侧问卷或补充说明。

## Agent作用

在该 Workflow 中，Agent 负责辅助整理和组织信息，而不是作出诊断判断。

Agent 的作用包括：

- 识别诊前信息中的关键字段。
- 将非结构化描述转化为医生可读摘要。
- 发现信息缺口并生成补充问题。
- 将近期健康反馈和历史资料放入同一上下文。
- 为诊中辅助决策 Workflow 准备输入。

## 医生角色

医生是诊前信息的最终使用者和判断者。

医生需要：

- 查看并确认诊前摘要是否准确。
- 判断哪些信息与本次诊疗相关。
- 决定是否需要进一步追问、检查或调整诊疗重点。
- 在诊中继续接管临床判断。

## 输出结果

- 诊前患者摘要。
- 本次就诊关键问题列表。
- 缺失信息或待补充问题。
- 近期健康状态概览。
- 可传递给诊中辅助决策 Workflow 的患者上下文。

## 与其他Workflow关系

- 与 [[诊中辅助决策 Workflow]] 的关系：诊前信息采集 Workflow 生成诊中所需的患者上下文，帮助医生更快进入临床判断。
- 与 [[诊后医嘱执行 Workflow]] 的关系：诊前采集可引用上一次诊后反馈，例如医嘱执行情况、随访结果和复诊原因。
- 与 [[连续照护闭环]] 的关系：它是闭环起点，将患者信息带入医生工作流。

## 备注

本 Workflow 不承担自动诊断职责，只负责帮助医生在接诊前获得更完整、更清晰、更连续的患者信息。
