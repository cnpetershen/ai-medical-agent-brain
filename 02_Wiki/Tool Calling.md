# Tool Calling

## 定义

Tool Calling 是 [[医生AI Copilot]] 中让 [[医疗AI Agent]] 调用外部医疗系统、数据接口和任务工具的能力。

在本项目中，它用于连接患者信息、病历记录、检查结果、医嘱任务和随访管理工具，使 [[Agentic Workflow]] 能够执行具体动作。

## 在医疗Copilot中的作用

- 连接外部医疗系统或模拟数据源。
- 查询患者基础信息、历史病历、检查结果和任务状态。
- 创建或更新医嘱执行任务、提醒和随访记录。
- 将 AI 输出从“文本建议”转化为医生可审核的流程动作。
- 为日志追溯和医生确认提供工具调用记录。

## 为什么项目需要

本项目要打造诊前、诊中、诊后的 [[连续照护闭环]]，不能只停留在对话层。

Tool Calling 让 Copilot 能够与医疗数据接口和任务系统协同：诊前查信息，诊中取检查结果，诊后生成和跟踪任务。没有 Tool Calling，Workflow 很容易变成静态文案，无法体现真实流程能力。

## 支撑哪个Workflow

- [[诊前信息采集 Workflow]]：调用患者信息、历史病历和问卷数据接口。
- [[诊中辅助决策 Workflow]]：调用检查结果、病历资料、知识检索或摘要生成工具。
- [[诊后医嘱执行 Workflow]]：调用任务生成、提醒、随访记录和状态更新工具。

## 输入输出

输入：

- 医生指令或 Workflow 当前步骤。
- 患者身份或就诊上下文。
- 需要查询、创建、更新或提醒的任务参数。
- 医疗系统、模拟数据源或知识库接口。

输出：

- 查询到的患者数据、病历片段或检查结果。
- 创建或更新后的任务状态。
- 提醒、随访或医嘱执行记录。
- 工具调用结果和异常信息。
- 需要医生审核的操作结果。

## 与其他知识节点关系

- [[医疗AI Agent]]：由医疗AI Agent 判断何时调用工具。
- [[Agentic Workflow]]：Tool Calling 发生在具体 Workflow 步骤中。
- [[RAG]]：RAG 可通过 Tool Calling 连接知识库或检索服务。
- [[Memory]]：工具调用结果可写入 Memory，成为连续照护上下文。
- [[Human-in-the-loop]]：关键工具动作需要医生审核或确认。
- [[医生AI Copilot]]：Tool Calling 是 Copilot 从辅助信息走向流程执行的关键能力。

## 备注

Tool Calling 不代表 AI 可以自动完成医疗决策。凡涉及诊疗判断、医嘱变更、风险处理的动作，都必须保留医生确认机制。
