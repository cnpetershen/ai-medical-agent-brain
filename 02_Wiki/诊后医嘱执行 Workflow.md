# 诊后医嘱执行 Workflow

## 定义

诊后医嘱执行 Workflow 是 [[医生AI Copilot]] 在医生完成诊疗后运行的任务跟踪与患者管理流程。

它通过 [[Agentic Workflow]] 将医生确认的医嘱、用药、康复、复诊和随访要求转化为可执行、可反馈、可追踪的管理任务，支撑 [[连续照护闭环]] 的延续。

## 临床场景

患者离开诊室后，医疗服务并没有结束。用药是否完成、康复任务是否执行、是否按时复诊、异常情况是否被发现，都会影响连续照护质量。

该 Workflow 面向的场景包括：

- 医生开具医嘱后，需要形成患者可执行的任务。
- 慢病或复诊患者需要持续随访和状态反馈。
- 患者用药、康复、检查复查存在遗忘或依从性不足。
- 医生需要了解患者诊后执行情况，为后续复诊做准备。

## 解决的问题

- 医嘱离开诊室后缺少持续跟踪。
- 患者可能无法准确理解或按时执行医嘱。
- 复诊和随访任务容易遗漏。
- 医生难以及时了解患者诊后状态变化。
- 诊后反馈无法有效回流到下一次诊前和诊中流程。

## 工作流程

1. 接收诊中辅助决策 Workflow 中由医生确认的医嘱和管理事项。
2. 将医嘱拆解为用药、康复、复查、复诊、随访等任务。
3. 为患者生成可理解的执行提醒和任务计划。
4. 持续记录患者执行状态和健康反馈。
5. 对未完成、异常或风险信号进行标记。
6. 必要时生成医生查看提醒或随访任务。
7. 将诊后反馈沉淀为下一次诊前信息采集的输入。

## 输入数据

- 医生确认后的医嘱。
- 用药计划、康复任务、复查要求和复诊时间。
- 患者执行反馈。
- 患者健康数据，例如血压、血糖、症状变化。
- 随访记录或异常情况反馈。
- 诊中形成的诊疗摘要和管理重点。

## Agent作用

在该 Workflow 中，Agent 负责将医生确认的医疗计划转化为可执行任务，并跟踪执行状态。

Agent 的作用包括：

- 拆解医嘱为具体任务。
- 生成患者侧提醒和随访事项。
- 记录任务完成状态和健康反馈。
- 识别未完成任务、异常反馈或复诊需求。
- 将诊后信息回流给医生和下一次诊前流程。

## 医生角色

医生是医嘱制定者、风险确认者和后续管理决策者。

医生需要：

- 确认诊后任务是否符合医嘱。
- 查看异常情况或高风险提醒。
- 决定是否需要调整医嘱、安排复诊或追加随访。
- 在下一次诊疗中使用诊后反馈作为判断依据。

## 输出结果

- 患者侧医嘱执行任务。
- 用药、康复、复查和随访提醒。
- 任务完成状态记录。
- 异常情况或风险提示。
- 可回流到诊前信息采集 Workflow 的诊后反馈摘要。

## 与其他Workflow关系

- 与 [[诊中辅助决策 Workflow]] 的关系：诊后医嘱执行 Workflow 接收医生在诊中确认的医嘱和管理任务。
- 与 [[诊前信息采集 Workflow]] 的关系：诊后执行反馈会成为下一次诊前信息采集的重要输入。
- 与 [[连续照护闭环]] 的关系：它让诊疗结果不止停留在诊室内，而是进入持续执行、反馈和复诊准备。

## 备注

本 Workflow 不改变医生医嘱，不自动作出新的医疗决策。它的核心职责是执行跟踪、反馈回流和辅助医生持续管理患者。
